🔹 UNIT I & II – 3 MARK QUESTIONS

1️⃣ Define Operating System (Very frequently asked – Unit I)
  1. An Operating System (OS) is a system software.
  2. It acts as an interface between user and hardware.
  3. It manages CPU, memory, I/O devices, and files.
  4. It controls execution of application programs.
  5. It provides an environment for program execution.
  6. It ensures efficient utilization of system resources.
  7. It handles process scheduling and synchronization.
  8. It provides security and protection mechanisms.
  9. It supports multitasking and multiprogramming.
  10. Examples include Windows, Linux, UNIX, macOS.
2️⃣ Define Semaphore (Very important – Unit II)
  1. A semaphore is a synchronization tool.
  2. It is used to control access to shared resources.
  3. It is an integer variable.
  4. It is accessed only through atomic operations.
  5. Two main operations are wait() and signal().
  6. wait() decreases the semaphore value.
  7. signal() increases the semaphore value.
  8. Semaphores prevent race conditions.
  9. They help in process synchronization.
  10. Types include binary semaphore and counting semaphore.
3️⃣ What is Thread (Unit I – Multithreading)
  1. A thread is the smallest unit of execution within a process.
  2. It is also called a lightweight process.
  3. A thread shares the code, data, and resources of its parent process.
  4. Each thread has its own program counter.
  5. Each thread maintains its own register set.
  6. Threads have their own stack for execution.
  7. Multiple threads can exist within a single process.
  8. Threads enable concurrent execution within the same process.
  9. Thread creation is faster than process creation.
  10. Threads improve CPU utilization and responsiveness.
4️⃣ What is Critical Section (Unit II – Synchronization)
  1. Critical section is part of a program.
  2. It accesses shared variables/resources.
  3. Only one process can execute it at a time.
  4. Prevents race conditions.
  5. Requires mutual exclusion.
  6. Must satisfy progress condition.
  7. Must avoid starvation.
  8. Used in concurrent systems.
  9. Controlled using semaphores or monitors.
  10. Essential for data consistency.
5️⃣ What is Deadlock (Unit II)
  1. Deadlock is a system state.
  2. Processes wait indefinitely.
  3. Each process holds a resource.
  4. Each process waits for another resource.
  5. No process can proceed.
  6. Resources are permanently blocked.
  7. Occurs in concurrent systems.
  8. Causes system halt.
  9. Requires special handling.
  10. Common in multiprogramming systems.

🔹 UNIT I & II – 8 MARK QUESTIONS

1️⃣ Explain Operating System and its Services (Very frequently asked – 8 marks)
  1. An Operating System is system software that acts as an interface between the user and computer hardware.
  2. It manages all hardware resources such as CPU, memory, and I/O devices.
  3. The OS provides an environment for the execution of programs.
  4. Program execution service loads a program into memory and executes it.
  5. I/O operation service allows programs to perform input and output operations safely.
  6. File system manipulation service enables creating, deleting, reading, and writing files.
  7. Communication service allows processes to exchange information using shared memory or message passing.
  8. Error detection service detects hardware and software errors.
  9. Resource allocation service allocates CPU, memory, and devices to processes.
  10. Accounting service keeps track of resource usage by users and processes.
  11. Protection service ensures controlled access to system resources.
  12. Security service prevents unauthorized access to the system.
  13. OS services improve user convenience.
  14. OS services improve system efficiency.
  15. Thus, OS acts as both a resource manager and a service provider.
2️⃣ Explain Critical Section Problem and Its Solutions (Very important – Unit II)
  1. Critical section is a part of code accessing shared resources.
  2. Only one process should execute critical section at a time.
  3. Mutual exclusion is mandatory.
  4. Progress condition must be satisfied.
  5. Bounded waiting ensures fairness.
  6. Race conditions occur if synchronization fails.
  7. Software solutions exist for critical section problem.
  8. Peterson's solution solves two-process synchronization.
  9. Hardware solutions use atomic instructions.
  10. Test-and-Set is a hardware instruction.
  11. Semaphores provide a general solution.
  12. Binary semaphores ensure mutual exclusion.
  13. Counting semaphores manage multiple resources.
  14. Monitors provide high-level synchronization.
  15. Proper synchronization ensures data consistency.
3️⃣ Explain CPU Scheduling – Criteria and Algorithms (Highly important – numericals + theory)
  1. CPU scheduling determines which process gets CPU control.
  2. It is essential in multiprogramming systems.
  3. CPU utilization should be maximized.
  4. Throughput should be increased.
  5. Turnaround time should be minimized.
  6. Waiting time should be minimized.
  7. Response time is critical in interactive systems.
  8. FCFS scheduling executes processes in arrival order.
  9. SJF scheduling selects the shortest burst time process.
  10. Priority scheduling assigns priority to each process.
  11. Round Robin scheduling uses time quantum.
  12. Multilevel queue scheduling uses multiple ready queues.
  13. Multilevel feedback queue allows movement between queues.
  14. Scheduling prevents CPU starvation.
  15. Proper scheduling improves overall system performance.

🔹 UNIT I & II – 15 MARK QUESTIONS

1️⃣ Explain Operating System – Functions, Goals and Types (Very frequently asked – 15 marks)
  1. An Operating System (OS) is system software that manages computer hardware and software resources.
  2. It acts as an interface between the user and the hardware.
  3. One of the main functions of an OS is process management.
  4. It creates, schedules, and terminates processes.
  5. Memory management is another key function of the OS.
  6. The OS allocates and deallocates main memory efficiently.
  7. File management allows creation, deletion, and manipulation of files.
  8. I/O device management controls and coordinates device operations.
  9. The OS provides protection and security to system resources.
  10. The goal of an OS is convenience, making the system easy to use.
  11. Another goal is efficiency, ensuring optimal use of resources.
  12. Batch operating systems execute jobs in batches.
  13. Time-sharing systems allow interactive use by multiple users.
  14. Real-time operating systems respond within strict time limits.
  15. Thus, the OS acts as a resource manager and service provider.
2️⃣ Explain Deadlock – Characterization and Handling Methods (Most important Unit II question)
  1. Deadlock is a situation where processes wait indefinitely.
  2. Each process holds at least one resource.
  3. Mutual exclusion must exist.
  4. Hold and wait condition must exist.
  5. No preemption must exist.
  6. Circular wait must exist.
  7. All four conditions must hold simultaneously.
  8. Deadlock prevents system progress.
  9. Deadlock prevention breaks a condition.
  10. Deadlock avoidance ensures safe state.
  11. Banker's algorithm is used for avoidance.
  12. Detection allows deadlock to occur.
  13. Wait-for graph detects deadlock.
  14. Recovery resolves deadlock.
  15. Deadlock handling is essential for system stability.

1️⃣ FCFS (First Come First Serve) Scheduling
Problem:
Process Arrival Time Burst Time
P1 0 6
P2 2 4
P3 4 5
Gantt Chart: 0 6 10 15 | P1 | P2 | P3 | Waiting Time (WT): WT(P1) = 0 WT(P2) = 6 - 2 = 4 WT(P3) = 10 - 4 = 6 Average WT = (0 + 4 + 6) / 3 = 3.33 Turnaround Time (TAT): TAT(P1) = 6 - 0 = 6 TAT(P2) = 10 - 2 = 8 TAT(P3) = 15 - 4 = 11 Average TAT = (6 + 8 + 11) / 3 = 8.33
2️⃣ Round Robin Scheduling
Problem: Time Quantum = 3
Process Burst Time
P1 10
P2 5
P3 8

3️⃣ Banker's Algorithm (Safety Check)
Given:
Allocation Matrix: P1: 0 1 0 P2: 2 0 0 P3: 3 0 2 Max Matrix: P1: 7 5 3 P2: 3 2 2 P3: 9 0 2 Available: 3 3 2 Need = Max - Allocation: P1: 7 4 3 P2: 1 2 2 P3: 6 0 0 Safe Sequence Check: 1. P2 can execute (Need ≤ Available) 2. After P2: Available = 3+2, 3+0, 2+0 = 5 3 2 3. P1 can execute (Need ≤ Available) 4. After P1: Available = 5+0, 3+1, 2+0 = 5 4 2 5. P3 can execute (Need ≤ Available) Safe Sequence: P2 → P1 → P3 ✔ System is in SAFE STATE
